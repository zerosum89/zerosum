# 오딘 실제 패치일 추론 규칙 v81

## 목적
오딘 패치노트에서 연도 오탐을 방지하기 위해, 본문 중간의 이벤트/시즌 날짜가 아니라 원문 게시 시점과 제목/상단 안내의 월·일만 사용한다.

## 우선순위
1. 원문 제목 또는 상단 제목의 `M/D(요일) 업데이트 상세 내역 안내`에서 월/일을 추출한다.
2. 제목이 `4월 4주차 업데이트`처럼 주차형으로 월/일이 없는 경우, 본문 상단의 `M/D(요일) 정기 점검 일정 안내`를 사용한다.
3. 위 둘이 없을 경우 본문 도입부의 `M월 D일(요일) ... 업데이트 점검`을 사용한다.
4. 연도는 `작성시간 YY.MM.DD`를 기준으로 앵커링한다. 단, 게시일이 12월 말이고 업데이트 월/일이 1월 초인 경우에는 가장 가까운 날짜가 되도록 다음 해를 허용한다.
5. 본문 중간의 이벤트 기간, 시즌 시작일, 보상 지급일, 판매 종료일, 후속 점검일은 실제 패치일 추론에 사용하지 않는다.

## 출력 필드
- `실제 패치일`: YYYY-MM-DD
- `항목명`: YY.MM.DD | 업데이트
- `패치노트 제목`: YY.MM.DD | 업데이트
