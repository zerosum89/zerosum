# v76 일반 패치 요약 표준 생성 규칙

## 목적

`일반 패치 요약`은 `본문 요약`의 세부 문장을 다시 설명하는 문장이 아니라, 패치노트 카드와 상세 패널에서 빠르게 필터링·스캔할 수 있는 상위 분류 태그입니다.

v76은 v65의 결과를 정답으로 복제하지 않습니다. 현재 `본문 요약`이 확정한 의미 카테고리를 기준으로, 신규 패치노트에도 동일하게 적용 가능한 규칙을 표준화합니다.

## 입력

1. `본문 요약` 배열
2. 선택 입력: `주요 업데이트 요약`, `대표 핵심 신호`
3. 사용하지 않음: DB1/DB2 레거시 필드, v65 캐시 값

## 출력

각 업데이트 이벤트에 `general_patch_summary` 1~3개를 생성합니다. 출시 이벤트는 생성하지 않습니다.

## 표준 라벨

1. 클래스/전투 갱신
2. 콘텐츠/시스템 조정
3. 서버/월드 조정
4. 성장/장비 갱신
5. 경제/보상 조정
6. 편의성 개선
7. 이벤트/보상 갱신
8. 상점/BM 갱신
9. 버그 수정

## 본문 요약 prefix 매핑

| 본문 요약 prefix | 일반 패치 요약 라벨 |
|---|---|
| 전투/밸런스, 클래스/스킬, 클래스/직업 | 클래스/전투 갱신 |
| PvE 콘텐츠, PvP/전쟁, 길드/전장, 길드/소셜, 반복 플레이, 운영/안정성, 플랫폼/접속 | 콘텐츠/시스템 조정 |
| 서버/월드 | 서버/월드 조정 |
| 성장/장비, 성장/외형, 성장/이벤트 | 성장/장비 갱신 |
| 경제/보상, 경제/거래 | 경제/보상 조정 |
| 편의/UI, 채팅/편의, 편의/제작 | 편의성 개선 |
| 이벤트/보상 | 이벤트/보상 갱신 |
| 상점/BM | 상점/BM 갱신 |
| 버그 수정 | 버그 수정 |

## 선택 규칙

1. `본문 요약` 각 줄의 prefix를 읽습니다.
2. prefix를 표준 라벨로 변환합니다.
3. 같은 라벨이 여러 번 나오면 점수를 합산합니다.
4. 본문 요약 앞쪽에 있는 항목일수록 소폭 가중합니다.
5. `서버/월드 조정`, `경제/보상 조정`은 실제 운영 영향이 커서 소폭 가중합니다.
6. 아래와 같은 일반 문장은 낮은 점수를 부여합니다.
   - 출석, 미션, 교환소 등 기간제 이벤트
   - 신규 패키지, 소환 상품, 시즌 패스
   - 전투, UI, 퀘스트, 보상 지급 등 주요 오류
7. 점수와 우선순위를 기준으로 최대 3개 라벨만 남깁니다.
8. `주요 업데이트 요약`이 있어도 일반 패치 요약을 비우지 않습니다. 두 정보는 역할이 다르기 때문입니다.

## 역할 분리

| 항목 | 역할 |
|---|---|
| 주요 업데이트 | 대형 변화, 신규 콘텐츠, 신규 성장축, 서버/경제 구조 변화 등 headline |
| 본문 요약 | 해당 패치의 구체적 변경 내용을 2~4줄로 설명 |
| 일반 패치 요약 | 해당 패치가 어느 범주에 속하는지 보여주는 상위 분류 태그 |

## 신규 패치 워크플로우

```txt
원문 URL 수집
→ 원문 본문 정제
→ 본문 요약 생성
→ 본문 요약 prefix 기반 일반 패치 요약 생성
→ 주요 업데이트 신호 생성
→ Patchnote View Model / HTML export
```
